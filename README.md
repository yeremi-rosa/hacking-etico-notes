# Ethical Hacking — Lab Notes

Public portfolio documentation for hands-on ethical hacking and cybersecurity labs.

This repository focuses on practical execution: lab setup, commands, results, findings, troubleshooting, scripts, and defensive takeaways. Each writeup is written so another student, junior analyst, or recruiter can understand what was tested, what happened, and what security lesson came from it.

---

## What This Repository Contains

| Type | Description |
|---|---|
| `phase-XX-*/` | Completed lab writeups and results organized by topic |
| `tools/` | Quick references for tools, flags, and workflows used in labs |
| `scripts/` | Small scripts created to automate or support lab work |
| `TEMPLATE.md` | Standard format used for lab documentation |
| `PUBLICATION_RULES.md` | Public documentation quality and safety standards |

---

## Current Lab Areas

| Area | Focus |
|---|---|
| Lab Environment | Virtual machines, isolated networking, snapshots, and repeatable setup |
| Reconnaissance | Passive, semi-passive, and active information gathering in authorized labs |
| Vulnerability Analysis | Service enumeration, CVE research, severity scoring, and validation |
| Exploitation Basics | Controlled exploitation inside intentionally vulnerable environments |
| Web Security | Common web vulnerabilities, testing workflow, and defensive notes |
| Active Directory | Windows domain security concepts in an isolated lab environment |
| Network Attacks | Local lab network attacks, packet analysis, and detection opportunities |
| Cloud Security | AWS identity, networking, logging, and security review concepts |
| Detection and Defense | Logs, monitoring, alerting, mitigation, and hardening takeaways |

---

## Each Lab Writeup Includes

- Objective
- Lab environment
- Commands executed
- Results and sanitized output
- Screenshots when useful
- Findings and technical explanation
- Troubleshooting notes
- Defensive takeaway: detection, mitigation, or hardening

---

## Safety and Ethics

All work is performed only in owned, intentionally vulnerable, or explicitly authorized lab environments. The goal is to learn offensive techniques responsibly and connect them to defensive security practices.
